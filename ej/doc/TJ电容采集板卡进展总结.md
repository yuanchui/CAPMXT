# TJ 电容采集板卡 — 项目进展与状态总结

> **文档用途**：对内掌握进展、对外汇报与演示配套说明  
> **需求依据**：[TJ电容采集板卡需求.pdf](./TJ电容采集板卡需求.pdf)  
> **编制日期**：2026-06-23  
> **当前版本**：固件主工程 Core + USB_DEVICE；上位机 Serial Terminal **v1.0.53**

## 一、执行摘要

本项目面向 **64×64 电容传感阵列** 的采集与可视化需求，当前已交付一套 **可演示、可量产调参** 的 **USB 桥接采集方案**：

| 维度 | 当前状态 | 一句话说明 |
|------|----------|------------|
| **硬件平台** | 已验证 | STM32F103 桥接板 + Microchip **mXT640UD** 触摸控制器 |
| **传感规模** | 阶段性 | 芯片物理矩阵 **32×20（640 节点）**，上位机稳定显示 **16×16（256 点）** |
| **通信接口** | 已达成 | **USB 2.0 Full Speed** 虚拟串口（CDC），PC 即插即用 |
| **数据输出** | 已达成 | Mode3 二进制协议（`AA 10 33`）、SPI 高速原始流、I2C 诊断帧 |
| **实时显示** | 已达成 | 热力图、3D 曲面、单点变化曲线、帧率/扫描频率统计 |
| **相对需求差距** | 待规划 | 目标 **64×64 / 500–1000 Hz / 4096 点力校准 API** 尚未达到 |

**结论**：已完成需求文档中「短期可接受方案」方向的核心验证（USB 通信、阵列变化量采集、按压位置与力度趋势感知、上位机可视化）。距离最终 **64 通道 × 64 通道、千赫兹级扫描** 的全规格产品，需在传感器选型、通道扩展与扫描架构上进入下一阶段。

## 三、当前技术方案

### 3.1 系统架构

![系统架构示意图](images/tj_system_arch.png)

**角色划分**：

- **mXT640UD**：完成电容扫描、信号处理，经 I2C 提供配置与 T37 诊断数据，经 SPI 调试口输出高速原始流。
- **STM32F103**：USB 与 PC 通信、I2C 配置触摸芯片、SPI 从机抓取调试数据、合成虚拟 SSN 帧边界、打包 Mode3 协议上传。
- **Serial Terminal（PC）**：矩阵可视化、协议解析、配置烧录（xcfg/enc）、产测与演示。

### 3.2 硬件概要

| 项目 | 规格 |
|------|------|
| MCU | STM32F103C8，72 MHz，64 KB Flash / 20 KB SRAM |
| 触摸芯片 | mXT640UD-CCUBHA1（Family `0xA6`，Variant `0x15`） |
| 传感矩阵 | 32 × 20 = **640 节点**（以 `INFO` 命令读回为准） |
| USB | Full Speed CDC 虚拟串口，64 B 最大包长 |
| 关键接口 | I2C2（配置）、SPI1 从机 RX-only（调试口）、CHG 中断（触摸事件） |
| 虚拟 SSN | 监视 MISO（PA10），在 PA9 合成帧边界，实现 SPI 帧提取 |

### 3.3 固件能力

固件工程：**STMUSBATMXT640**（主路径 `Core/` + `USB_DEVICE/App/mxt/`）

| 能力模块 | 说明 | 状态 |
|----------|------|------|
| I2C 诊断 | `START MAP16*` 定时读 T37，输出 Mode3 包 | 稳定 |
| SPI 高速流 | `SPISTART` / `SPISTART1` / `SPISTART3` 三种输出模式 | 稳定 |
| 虚拟 SSN | 帧内 DMA 全速采集、帧间 USB 发送的半流水线 | 稳定 |
| 配置烧录 | CFGWRITE / ENCWRITE，支持 xcfg 与固件更新 | 稳定 |
| mxt-app 桥接 | 二进制 I2C 读写、引脚读取，兼容产线工具链 | 稳定 |
| CHGNO 扩展 | Mode3 包附带触摸点坐标与动作类型 | 稳定 |
| test-V1.7 变体 | 硬件 NSS + IT 逐字节，适配 640U 三次硬件 SSN | 并行维护 |

**推荐对外演示命令**：

```text
SPISTART3          # SPI 高速路径，16×16 Mode3 二进制，适合实时矩阵
START MAP16L90X 10 # I2C 诊断路径，10 ms 间隔，适合对比验证
SPIDBG             # 查看 SSN/SPI/USB 队列与丢帧计数
```

### 3.4 数据协议（对外接口摘要）

#### Mode3 标准包（40 字节）

```text
AA 10 33 | LEN(40) | FRAME_ID | ROW_ID(0~15) | DATA[32] | CRC16
```

- 每完整帧 16 行，每行 16 个采样点（各 2 字节，大端 int16）
- CRC16-CCITT-FALSE（多项式 `0x1021`，初值 `0xFFFF`）
- 详见：[Mode3_Output_Protocol.md](./Mode3_Output_Protocol.md)

#### SPI 原始流包（SPISTART）

```text
88 77 66 | LE u16 长度 | payload(1~514 B)
```

#### 性能参考（SPISTART3 稳定运行）

| 指标 | 典型值 |
|------|--------|
| 触摸扫描帧率 | 约 20–40 Hz（DEBUG 模式，随 xcfg 与负载变化） |
| 每帧 USB 数据量 | 640 B（16 包 × 40 B Mode3） |
| USB 吞吐 | 约 19 KB/s @ 30 Hz |

### 3.5 上位机软件（Serial Terminal）

| 项目 | 说明 |
|------|------|
| 版本 | **v1.0.53** |
| 技术栈 | Electron + 串口/WebUSB |
| 核心功能 | 16×16 热力图、3D 曲面、帧率/变化频率曲线、矩阵对比、xcfg/enc 烧录 |
| 数据格式 | Mode3 二进制、HEX 文本、SPI 原始流 |

## 四、已验证能力（可对外演示）

以下能力对应需求文档 **§1.4 视频演示效果**，当前均可复现：

1. **单点电容变化感知**  
   手指按压阵列中某一点，对应矩阵格变化量增大/减小，3D 曲面该位置隆起或凹陷。

2. **力度趋势区分**  
   按压力度变化时，变化量数值随之变化，可通过色阶与 3D 高度直观对比。

3. **多点/区域按压**  
   手掌或大面积按压时，多个相邻格同时响应，可观察受力区域分布。

4. **实时刷新与统计**  
   界面显示帧率、矩阵扫描频率、帧号；支持历史频率曲线。

5. **配置与产线能力**  
   支持 xcfg 配置导入导出、固件 ENC 烧录、I2C 地址扫描、诊断命令交互。

**建议演示流程**（约 5 分钟）：

1. USB 连接 → 上位机识别 COM 口 → `INFO` 确认芯片与矩阵尺寸  
2. 发送 `SPISTART3` 或 `START MAP16L90X 10` → 打开矩阵/3D 视图  
3. 单点按压 → 展示热力图与 3D 响应  
4. 渐变施力 → 展示数值趋势  
5. 大面积按压 → 展示区域分布

## 五、差距分析与风险

### 5.1 与全规格需求的主要差距

| 差距项 | 需求目标 | 现状 | 影响 |
|--------|----------|------|------|
| 阵列规模 | 64×64（4096 点） | 16×16 显示（256 点） | 无法满足大面积高密度传感 |
| 通道架构 | CD64 + CS64 独立 | 单 mXT640（32×20） | 需多芯片或专用 ASIC/FPGA 扩展 |
| 扫描频率 | 500–1000 Hz | 20–40 Hz（DEBUG 模式） | 高速动态场景受限 |
| 精度指标 | 0.005–0.01 pF | 未出具测试报告 | 无法对外承诺精度指标 |
| 力校准 API | 4096 点标准化接口 | 256 点，协议已通但规模不足 | 需扩展矩阵与接口定义 |
| 归一化范围 | 0–255 / 0–1023 | 上位机内部 0–100 映射 | 协议层可增加统一缩放字段 |

### 5.2 技术风险与约束

1. **DEBUG 模式降速**：开启 SPI 调试口（DEBUGCTRL2）会显著降低触摸芯片扫描速率，属芯片机制限制。  
2. **SRAM 瓶颈**：STM32F103 仅 20 KB RAM，SPI 采用双槽帧队列 + 背压丢帧策略；提高帧率需更强 MCU 或外扩缓冲。  
3. **USB FS 带宽**：64 B 包长限制下，64×64 全矩阵千赫兹上传需 USB HS 或数据压缩/差分编码。  
4. **传感器绑定**：当前方案与 mXT640 生态绑定，迁移至纯电容 ASIC 需重新设计硬件与协议。

## 七、建议后续路线

按需求文档「短期可先 50 Hz、8/16 通道」的指引，建议分三阶段推进：

### 阶段 A — 夯实当前 16×16 方案（1–2 个月）

- [ ] 出具电容变化量与按压力度的标定报告（至少 16×16 范围内）  
- [ ] 优化 xcfg，在不开 DEBUG 口时评估 I2C 路径最高扫描频率  
- [ ] 协议层增加统一归一化字段（0–255 / 0–1023 可选）  
- [ ] 完善对外 API 文档（C/Python 解析示例）

### 阶段 B — 扩展通道与矩阵（2–4 个月）

- [ ] 评估多 mXT 级联或更大矩阵芯片（如 32×32 及以上）  
- [ ] 硬件 FPC 0.5 mm 接口标准化  
- [ ] MCU 升级（如 F4/F7 + USB HS）以支撑更大数据量

### 阶段 C — 对标全规格 64×64（中长期）

- [ ] CD64 + CS64 双通道采集架构设计  
- [ ] 500–1000 Hz 扫描方案（专用采集 IC 或 FPGA）  
- [ ] 4096 点力校准算法与产线标定流程  
- [ ] 精度 0.005–0.01 pF 测试与认证
